# Introduction
Today, I will be building a SIEM lab using Splunk and Windows Server 2022. This project will allow me to imitate real security threats and learn how to detect them.  

## Importance
The importance of Splunk is to make monitoring a large amount of logs and alerts easier, so that security professionals can spot threats and deal with them faster. The visualization and threat detection make it perfect for overseeing a large environment. 

## Prerequisites
1. Windows Server Machine/VM
2. Splunk Free Edition

### Set-up
First, I installed the [`Windows Server 2022 ISO`](https://www.microsoft.com/en-us/evalcenter/download-windows-server-2022) on VirtualBox and then ran the basic installation of Splunk Enterprise (the indexer and GUI) and Splunk Universal Forwarders (forwards logs to the indexer).

## Basic Simulation
Now that the logs are set up correctly its time to trigger them. I'll pretend be an attacker and start with something simple like a ping to see if the server is there.

![Desktop View](..\screenshots\Screenshots\2026-02\VirtualBoxVM_mbrC8omRIm.png)

Although I tried to send ping packets to the Windows Server, I didn't get any response and got 100% packet loss, but they still showed up in the logs.

![Desktop View](..\screenshots\Screenshots\2026-02\VirtualBoxVM_8KHzPGnpcD.png)

In this picture, we can see what happened to the ping requests; they were dropped instead of being processed by the firewall. Firewalls will tend to do this to prevent DDOS attacks or unwanted scanning.

## What's Next?
Now that Splunk and its logging system works we will dive deeper into more realistic attacks, visualize how to filter logs for better visibility, and imitate the procedures of a security professional.
